# User Stories for Intervu.video

This document outlines the various user stories that encapsulate the primary use scenarios for the system. Each story is told from the perspective of a specific user role.

## 0. As a New or Returning User (Authentication & Onboarding)

- **Story:** As a **New User**, I want to **sign up for an account using my email and password or a social provider like Google** so that I can get access to the platform.
- **Story:** As a **New User**, I want to **choose my user type (e.g., 'Hiring' or 'Candidate') during registration** so that the system can tailor the experience for me.
- **Story:** As a **New Hiring User**, I want to **provide my organization's name during sign-up** so that my account and team space are created correctly.
- **Story:** As a **New User**, I want to **receive a verification email after registering** so that I can confirm my account is secure.
- **Story:** As a **Returning User**, I want to **log in easily with my credentials** so that I can quickly access my dashboard.
- **Story:** As a **User who forgot my password**, I want to **request a password reset link via email** so that I can regain access to my account.
- **Story:** As an **Invited User**, I want to **accept a team invitation through a unique link** so that I can join my organization's workspace without a complicated setup.

## 1. As an Administrator

- **Story:** As an **Admin**, I want to **invite new users to my organization and assign them roles** so that I can build my team and control their access levels from the start.
- **Story:** As an **Admin**, I want to **manage user roles and permissions centrally** so that I can ensure team members only have access to the features relevant to their job.
- **Story:** As an **Admin**, I want to **view a comprehensive audit trail of all major actions** so that I can monitor system activity for security and compliance purposes.
- **Story:** As an **Admin**, I want to **configure system-wide settings, such as integrating our own AI provider or storage solution**, so that I can customize the platform to our company's specific needs and policies.
- **Story:** As an **Admin**, I want to **deactivate or remove a user from the organization** so that I can manage team changes when an employee leaves the company.

## 2. As an Interviewer / Hiring Manager

- **Story:** As a **Hiring Manager**, I want to **create interview templates with custom video and text questions** so that I can standardize the screening process for specific job roles.
- **Story:** As a **Hiring Manager**, I want to **use pre-defined templates for common roles** so that I can quickly get started without building an interview from scratch.
- **Story:** As a **Hiring Manager**, I want to **schedule an interview and send an invitation link to a candidate** so that they can complete the video interview on their own time.
- **Story:** As a **Hiring Manager**, I want to **review a candidate's recorded video responses and see AI-generated analysis** so that I can efficiently screen candidates and identify top performers.
- **Story:** As a **Hiring Manager**, I want to **score submissions against a predefined rubric and leave notes** so that my evaluation is structured and consistent across all candidates.
- **Story:** As a **Hiring Manager**, I want to **view a dashboard of all ongoing and completed interviews** so that I can track the progress of all my candidates in one place.
- **Story:** As a **Hiring Manager**, I want to **create and manage different interviewer personas (avatars)** so that candidates can interact with an AI that fits our company culture.

## 3. As a Coordinator

- **Story:** As a **Coordinator**, I want to **view all scheduled interviews and their statuses** so that I can manage logistics and follow up on pending invitations.
- **Story:** As a **Coordinator**, I want to **reschedule an interview or extend a deadline for a candidate** so that I can accommodate requests and ensure a positive candidate experience.
- **Story:** As a **Coordinator**, I want to **send reminders to candidates who have not yet completed their interviews** so that we can maximize our completion rates.

## 4. As a Candidate

- **Story:** As a **Candidate**, I want to **receive a clear email invitation with a single link to start my interview** so that the process is simple and easy to begin.
- **Story:** As a **Candidate**, I want to **run a device check to test my camera and microphone before starting** so that I can be confident my equipment is working correctly.
- **Story:** As a **Candidate**, I want to **have a practice question to get comfortable with the recording process** so that I feel prepared for the actual interview.
- **Story:** As a **Candidate**, I want to **record my video answers for each question within a set time limit** so that I can complete the interview asynchronously.
- **Story:** As a **Candidate**, I want to **review and consent to the company's data privacy policy** so that I know how my information will be used.
- **Story:** As a **Candidate**, I want to **receive a confirmation after successfully submitting my interview** so that I have peace of mind that it was received.

## 5. As a Survey Respondent

- **Story:** As a **Survey Respondent**, I want to **access a survey through a simple link without needing to log in** so that I can provide feedback with minimal friction.
- **Story:** As a **Survey Respondent**, I want to **answer a series of text or video-based questions** so that I can provide rich, detailed feedback.
- **Story:** As a **Survey Respondent**, I want the **survey experience to be smooth and intuitive on both desktop and mobile devices** so that I can respond from anywhere.
